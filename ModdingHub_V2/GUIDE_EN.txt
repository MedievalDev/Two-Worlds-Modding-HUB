â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€
This is one of the hub's most useful features:

  The hub scans the guides/ folder every time it loads. Any .txt or
  .md file it finds that is NOT already a built-in guide gets added
  automatically to the guide list.

To add a new guide:
  1. Write your guide as a .txt or .md file
  2. Drop it into the guides/ folder
  3. Restart the hub (or switch tabs)
  4. Your guide appears with a green "ğŸ“‚ Auto" label

For bilingual guides, use the naming convention:
  my_guide_en.txt    â† English
  my_guide_de.txt    â† German

The hub pairs them automatically. If only one language exists,
it is used for both.

The guide title is generated from the filename:
  my_cool_guide_en.txt  â†’  "My Cool Guide"

The guides/ folder name is auto-detected. The hub checks for:
  guides/  Guids/  Guides/  guids/

FILE FORMAT DATABASE
â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€
The hub documents all Two Worlds 1 file formats. When you search
for a format extension (e.g. ".wd" or ".phx"), the tools that
handle that format are highlighted.

  .wd   â€” Mod archive (zlib, GUID). .phx must NOT be compressed!
  .lan  â€” Language file (16,194 translations, binary, UTF-16-LE)
  .par  â€” Parameter database (items, NPCs, skills, zlib-compressed)
  .lnd  â€” Level/map (terrain, objects, spawns, 128x128 tiles)
  .vdf  â€” 3D model (mesh, textures, animations)
  .phx  â€” Physics/collision (cook in editor, never compress!)
  .lhc  â€” LevelHeaders cache (regenerate after every map change)
  .idx  â€” Quest data (SOAP-XML export from WhizzEdit)
  .qtx  â€” Quest logic (plaintext, compiled for engine)
  .shf  â€” WhizzEdit project (.NET binary, read-only in our tools)
  .bmp  â€” Minimap image (one per map tile)
