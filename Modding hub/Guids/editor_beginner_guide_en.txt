OVERVIEW
â”€â”€â”€â”€â”€â”€â”€â”€
The TwoWorldsEditor is the official level editor for Two Worlds 1,
included in the TwoWorlds SDK version 1.3. It was the primary tool
used by Reality Pump Studios to build the game's open world, and it
is the only tool available for creating custom maps and mods for
Two Worlds 1.

The editor is a fully featured 3D level design tool that allows you
to sculpt terrain, paint textures, place objects, configure weather
and sky, set up NPC encounters, and design both surface and
underground levels. It also includes a console system for advanced
operations and batch processing.

This guide covers everything a new user needs to know to get started
with the editor, from installation to placing your first objects.
For exporting your map as a playable mod, see the companion guide
"TW1 Map to Mod Conversion Guide". For creating dungeons and caves,
see the "TW1 Dungeon Creation Guide (Part 1 & 2)".

MAP NAMING CONVENTION
â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€
Two Worlds uses a grid coordinate system for map tiles:

  Map_[Letter][Number].lnd
  Examples: Map_A01, Map_B05, Map_F03

Letters represent columns (East-West), numbers represent rows
(North-South). When creating mods that replace vanilla maps, your
files must use the exact same names as the originals.

For new custom maps used in standalone mods, you can use any valid
name following the Map_ prefix convention.

  SECTION 8 â€” STAMPS SYSTEM  (Advanced Brushes)
================================================================================

One of the editor's most unique features is the Stamp system. Stamps
allow you to use any 3D model from the game as a terrain brush. This
means you can stamp body shapes, rock formations, building outlines,
or any other mesh onto the terrain â€” both for sculpting and texturing.

ACCESS
â”€â”€â”€â”€â”€â”€
The Stamps section is in the bottom area of the Texture Set Manager.
There are 15 stamp slots, each corresponding to a number on the
Stamp selection pad in the main editor panel.

HOW TO SET UP A CUSTOM STAMP
â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€
  1. Open the Texture Set Manager (Setup textures set button).

  2. FIRST: Change the texture set name and save as a new set.
     This prevents overwriting an original game set.

  3. Find the "translateStampObject" field for the stamp slot you
     want to assign (00 through 14).

  4. Click the "..." button next to it to open the model file browser.

  5. Browse the game's model folders and select a .vdf file (the
     game's 3D model format). Any model can be used â€” characters,
     objects, environmental pieces, anything.

  6. Click Open to assign the model as a stamp.

  7. Save and Reload the texture set.

  8. Back in the editor, select the stamp number you assigned and
     start painting/sculpting. The brush will use the 3D model's
     shape.

EXAMPLE USE CASES
â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€
  â€¢ Use a rock formation model as a stamp to create natural-looking
    rocky terrain by stamping it across a hillside.
  â€¢ Use a path or road model to quickly lay down road shapes.
  â€¢ Use body or character models for... creative terrain sculpting.

The Specials folder in the stamp browser contains additional
predefined stamp shapes beyond the Default set.

Q: How do gates behave?

  â€¢ Outside populated areas: Gates stay closed, but the player can

  â€¢ In custom-built structures: Gates will most likely NOT auto-open.

â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€
  â€¢ Place chest objects from the Objects section.
  â€¢ Players can open any placed chest by lock-picking.
  â€¢ Loot content is ALWAYS randomly generated based on the player's
    current character level.
  â€¢ You do NOT control what specific items appear â€” only whether
    a chest exists at that location.


Q: How do I place working enemies?
â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€
There are two methods, but only one works properly:

  CORRECT: Use Spawner Markers
  Place spawner markers (found in the Markers section of the Objects
  browser). These generate living, active enemies at their location
  when the game runs.
    â€¢ Standard enemies: 2-6 spawn per marker
    â€¢ Tough enemies/giants: 1-3 spawn per marker

  WRONG: Place NPC Models Directly
  If you place an NPC character model directly, it will appear in-game
  but will be completely inert â€” it stands still like a statue. It
  won't move, patrol, or react unless directly attacked. The NPC's
  name will also display as generic/unusual text.

  ALWAYS use spawner markers for functional enemies.


Q: My objects are floating / sunk into the ground!
â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€
Use NumPad + and - to raise and lower objects. Some objects have

You can also use the console command to batch-adjust:

  editor.AddZToAll xxx
  (adds a height offset to all objects on the level)

After placing objects, always run Alt+U (Update Terrain Passing) and

  editor.joinlevels MapA MapB MapC MapD Joined.lnd
    Merge multiple levels into one (textures must be synchronized).
    Example: editor.joinlevels Map_E03.lnd Map_F03.lnd Map_E04.lnd
             Map_F04.lnd Joined_EF34.lnd

  Editor.SaveAllLevelsToTexture layer mode output.tga colors...
    Export level data as texture image for visualization.

  Editor.foreachlevel @script.txt
    Run a script on every level (surface + underground).

  Editor.foreachsurfacelevel @script.txt
    Run a script on every surface level.

  Editor.foreachundergroundlevel @script.txt
    Run a script on every underground level.

PHYSICS (Required before exporting as mod!)
â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€
  editor.cookphysx.mode geomipmap
  editor.cookphysx.strength = 1.0
  editor.cookphysx.overwrite = 1
  editor.cookphysx.pc out

  Run these four commands in sequence to generate physics data.
  The .phx file is written to Saves\Levels\Physic\.
  Must be done for EVERY level (surface and underground).
  The .phx file must NOT be compressed in the .wd archive.
